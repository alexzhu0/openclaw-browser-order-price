# Release 2026.03.25

## 概要

第四次升级，重点补齐补跑闭环，让多轮运行最终都能沉淀到一份最终结果。

## 主要内容

- 多账号模式新增独立的 `inputOffset` / `inputLimit`
  - 不再受 `runner.json` 单账号 `offset` / `limit` 污染
- 多账号跑完后自动导出待补跑清单：
  - `pendingRerunFile`
  - `pendingRerunExcelFile`
- 新增待补跑入口：
  - `npm run print-multi-pending-plan`
  - `npm run run-multi-pending`
- 待补跑默认筛选：
  - `relogin_required`
  - `checkout_blocked`
- 非交互模式下检测到登录页，结果明确标记为 `relogin_required`
- 多账号模式默认关闭交互式回车等待，避免某个 worker 卡死拖住整轮执行
- 新增最终结果闭环：
  - `finalMergedOutputFile`
  - `finalMergedExcelFile`
  - 无论跑几轮、补跑几轮，都会持续覆盖更新这份最终完整结果

## 相关脚本

- `scripts/multi_account_runner.mjs`
- `scripts/rerun_pending_multi.mjs`
- `config/multi_runner.json`

## 实测记录

- `behavior-2` 下，`20260325-1` 这轮里 `accountA` 在约 index 22 左右提示登录
- 调整到 `behavior-3` 后，实测不掉登录，稳定性更好

## 说明

- 这版的核心不是“再提高一次点击成功率”，而是把“失败后怎么补、补完怎么汇总”为一个完整流程
- 从这一版开始，项目更适合连续多日运行和逐轮补齐数据
